# 三分钟演示脚本

## 演示前

1. 启动 `backend/run_backend.bat`，确认 `/health` 返回 `UP`。
2. 安装 Debug APK，确认手机可以访问电脑的 8080 端口。
3. 准备一个已发布作品，避免现场网络影响完整闭环。

## 演示流程

### 0:00–0:30 用户与社区

- 登录账号，进入“发现”。
- 播放一首社区作品，展示迷你播放器。
- 进入讨论页，完成点赞和评论。

### 0:30–1:40 AI 创作

- 在“创作”选择一个灵感模板，说明提示词会自动带入。
- 设置风格、BPM 和时长并提交。
- 说明 Android 轮询的是自己的 Ktor 任务接口，模型 Key 不在 APK 中。
- 展示任务进度、失败提示和历史恢复能力。

### 1:40–2:20 发布闭环

- 试听生成音频。
- 点击“保存并发布”，返回社区查看作品。
- 切换另一个账号，证明作品是后端持久化数据而非本地假数据。

### 2:20–3:00 工程设计

- 展示 `MusicGenerationProvider`，说明 Fake/MiniMax/Replicate 的替换方式。
- 展示 `AiGenerationJobs`，解释任务状态和错误为什么能恢复。
- 展示作品管理、收藏和歌单，说明项目同时覆盖 AI 创作与音乐社区。

## 网络异常兜底

如果真实模型调用失败，明确说明切换到了 Fake Provider 或展示历史生成结果。不要把缓存作品伪装成刚生成；重点展示系统闭环和可替换架构。
